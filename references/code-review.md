## 一、代码质量深度检查

### 1.1 类型系统与架构

- [ ] **TypeScript类型完整性**：组件 Props、状态管理、函数返回值的类型定义
- [ ] **Any类型使用情况统计**：与 AI 替代建议
- [ ] **公共类型定义**：集中化管理程度
- [ ] **接口契约**：与后端 API 类型对齐度

### 1.2 组件设计与工程化

- [ ] **单一职责原则**：组件拆分合理性
- [ ] **AI代码识别**：分析 AI 生成的组件设计模式一致性
- [ ] **副作用封装**：自定义 Hooks 的质量与复用性
- [ ] **Hooks依赖数组**：准确性与完整性
- [ ] **组件边界定义**：模块耦合度

### 1.3 性能优化指标

- [ ] **AI提交行数统计**：标记 AI 生成的性能相关代码量
- [ ] **重渲染分析**：React.memo、useMemo、useCallback 使用合理性
- [ ] **虚拟滚动**：在大列表中的应用情况
- [ ] **代码分割策略**：React.lazy 与动态导入使用
- [ ] **资源优化**：图片压缩、Bundle 大小分析

### 1.4 可访问性与用户体验

- [ ] **语义化HTML标签**：使用规范
- [ ] **键盘导航支持**：完整度（无需检查）
- [ ] **ARIA属性**：应用准确性
- [ ] **色彩对比度**：WCAG 合规性

### 1.5 样式工程化

- [ ] **CSS模块化**：样式作用域管理
- [ ] **设计系统**：样式变量一致性
- [ ] **全局样式污染**：风险排查
- [ ] **响应式布局**：实现质量

### 1.6 安全与依赖管理

- [ ] **XSS防护**：输入验证与安全渲染
- [ ] **第三方依赖**：安全版本检查
- [ ] **AI生成代码安全扫描**：专门检查 AI 代码的安全漏洞

## 二、AI 代码专项分析

### 2.1 AI 代码识别与标记

**代码注释标记规范：**

```javascript
// @ai-generated-start
// 生成时间: YYYY-MM-DD
// 生成工具: GitHub Copilot/Cursor/其他
// 人工审核状态: [已审核/待审核]
// @ai-generated-end
```

**Git 提交信息标记规范：**

```bash
git commit -m "feat: xxx [AI]"
```

### 2.2 AI 代码质量门禁

- [ ] **类型安全**：AI 代码必须通过 TypeScript 严格模式
- [ ] **性能要求**：避免 AI 常见的过度抽象与性能陷阱
- [ ] **安全扫描**：AI 代码必须通过安全漏洞专项检测
